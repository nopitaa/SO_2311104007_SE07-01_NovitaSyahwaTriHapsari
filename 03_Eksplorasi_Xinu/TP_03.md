## Dasar Teori
Xinu adalah sistem operasi sederhana yang dikembangkan untuk tujuan pembelajaran, khususnya dalam memahami konsep dasar sistem operasi. Xinu memiliki struktur yang lebih kecil dan sederhana dibandingkan sistem operasi seperti Linux, sehingga lebih mudah dipelajari oleh mahasiswa.

Xinu dijalankan menggunakan Oracle VM VirtualBox yang memungkinkan satu komputer menjalankan beberapa sistem operasi melalui teknologi virtualisasi. Terdapat dua virtual machine yang digunakan, yaitu development system untuk melakukan pengembangan dan kompilasi source code, serta backend system untuk menjalankan sistem operasi Xinu.

Sebelum dijalankan, source code Xinu harus melalui proses compile menggunakan perintah make untuk menghasilkan file image sistem operasi. Setelah itu, backend VM melakukan proses booting melalui jaringan menggunakan protokol PXE untuk mengambil file image Xinu dari server. Interaksi dengan Xinu dilakukan melalui komunikasi serial menggunakan aplikasi Minicom, sehingga pengguna dapat memberikan perintah langsung kepada sistem operasi yang sedang berjalan.


## Guided
    - Berapa jumlah perintah pada Xinu? 
    18 perintah yang tersedia pada shell Xinu.
    - Sebutkan 2 perintah yang mempunyai fungsi yang sama! 
    exit dan logout itu perintah yang sama
    - Berapa IP address Xinu?  
    net0: 192.168.201.103/255.255.255.0
    - Perintah apa yang digunakan untuk mengetahui IP address? 
    ipaddr
    - Berapa IP DNS server yang digunakan oleh Xinu? 
    tftp://192.168.201.1/xinu.grub
    - Terdapat berapa proses yang sedang berjalan pada Xinu? 
    Pada sistem operasi Xinu terdapat sekitar 5 proses yang sedang berjalan setelah sistem selesai booting. Proses tersebut meliputi null process, main process, shell process, network process, dan device process.
    - Proses apa yang mempunyai prioritas paling rendah? 
    Proses yang memiliki prioritas paling rendah adalah Null process, karena proses ini hanya berjalan ketika tidak ada proses lain yang menggunakan CPU.
    - Proses apa yang mempunyai ukuran paling besar? 
    Proses yang biasanya memiliki ukuran paling besar adalah Main process, karena proses ini memanggil dan menjalankan berbagai fungsi awal sistem.
    - Proses apa yang berada dalam state current? 
    Proses yang berada dalam state current adalah Shell process, karena proses ini sedang aktif menerima perintah dari pengguna.
    - Proses apa yang berada dalam state suspend? 
    Proses yang berada dalam state suspend biasanya adalah Network process atau proses lain yang sedang menunggu event tertentu.
    - Berapa PID (Process ID) dari Main process?
    PID dari Main process pada Xinu yaitu 1.
