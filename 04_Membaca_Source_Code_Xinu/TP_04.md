## Dasar Teori
Xinu merupakan sistem operasi yang dirancang untuk sistem embedded dan mengikuti paradigma *cross-development*, di mana pengembang membuat serta mengompilasi kode pada komputer biasa (PC/laptop), kemudian mengunggah file image hasil kompilasi ke sistem target seperti mikrokontroler. Bahasa pemrograman yang digunakan adalah bahasa C karena sifatnya yang *low-level* sehingga dapat langsung memanipulasi perangkat keras.

Source code Xinu terorganisasi dalam beberapa direktori utama. Direktori `./compile` berisi Makefile untuk mengompilasi dan me-*link* image Xinu, sementara `./config` dan `./device` masing-masing menyimpan konfigurasi dan source code device driver yang mengendalikan perangkat keras. Direktori `./include` memuat header file yang dibutuhkan saat pengembangan, `./system` berisi source code kernel Xinu yang menangani manajemen resource dan penjadwalan, serta `./shell` berisi source code perintah-perintah shell.

Beberapa file penting dalam Xinu antara lain `xinu.h` sebagai master include file, `process.h` untuk deklarasi struktur tabel proses, `resched.c` sebagai penjadwal proses, serta `initialize.c` yang berisi fungsi inisialisasi sistem. Setiap *system call* pada Xinu umumnya berkorespondensi langsung dengan nama file yang sama di folder `./system`, misalnya syscall `resume` terdapat pada file `resume.c`. Xinu juga menggunakan konvensi penamaan tipe data berformat `nama_kegunaan` diikuti `ukuran_data` untuk menunjukkan ukuran dan kegunaan suatu variabel secara eksplisit.
